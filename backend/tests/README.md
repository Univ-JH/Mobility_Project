# backend/tests

백엔드 테스트 코드(구현 전 계획 문서/체크리스트 포함) 영역이다.

## 호환성
- 계약 테스트: MQTT payload schemaVersion 호환성, API 응답 포맷 계약을 기준으로 한다.
- 안전 테스트: 상태 전이 허용/금지 매트릭스가 깨지지 않음을 검증한다.

## 확장성
- 새로운 이벤트 타입/정책 룰이 생기면:
  - 정책 엔진 결정성 테스트
  - ingestion 멱등/순서역전 테스트
  - 응급 케이스 플로우 테스트

## 가시성/명확성
- 테스트 실패 원인이 곧바로 추적되도록 traceId/correlationId 기반으로 로그를 남긴다(테스트에서도).
