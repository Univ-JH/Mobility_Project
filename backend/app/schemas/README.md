# backend/app/schemas

요청/응답 DTO와 내부 이벤트/상태 표현을 위한 Pydantic 스키마(또는 타입) 정의 영역이다.

## 호환성
- MQTT payload(v1)와 API DTO는 최대한 동일한 필드명/타입을 사용한다.
- `schemaVersion` 변경 시 하위 호환을 고려해 스키마 버전을 분리한다.

## 확장성
- 새로운 이벤트 타입 추가 시:
  - `EventType` 정의
  - Event 저장/조회 DTO 확장
  - 상태 전이 검증 스키마 반영

## 가시성/명확성
- “문서에서 말하는 v1 필드”와 “스키마 코드”가 다르면 반드시 문서 업데이트 또는 스키마 수정으로 정리한다.
