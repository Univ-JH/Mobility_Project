# backend/app/domain

도메인 타입, 정책 DSL의 표현/결정적 평가 결과(Decision Object), 상태 전이(전이 매트릭스), 이벤트 타입 등을 정의하는 디렉터리다.

## 호환성
- 정책 DSL 규칙은 `12-policy-engine-dsl.mdc` 및 `BACKEND_IMPLEMENTATION_PLAN.md`의 DSL 섹션과 1:1로 대응한다.
- 상태 전이는 `26) 상태 전이 유효성 매트릭스`를 기반으로 “허용/금지/비정상 라벨링” 규칙을 고정한다.

## 확장성
- 새로운 이벤트 타입은:
  1. `EventType` enum/타입 추가
  2. 저장/조회 DTO에 반영
  3. 상태 전이 검증 로직에 반영

## 가시성/명확성
- 도메인 로직은 외부 I/O 없이 동작하는 순수 함수/클래스로 유지한다(테스트 용이성).
