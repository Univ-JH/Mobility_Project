# frontend-mobile

React Native 기반 모바일 앱 영역이다. 실시간 상태/응급 알림을 사용자에게 제공하고, 사용자 확인/취소 액션을 백엔드에 전달한다.

## 호환성
- 응급/알림 UX는 백엔드 API 계약과 `caseId` 식별자를 기준으로 동작한다.
- 이벤트 계약 필드는 v1 스키마(`severity/confidence/reason`)를 따른다.

## 확장성
- 실시간 스트림 방식이 바뀌어도 API 계층과 상태 계층을 통해 화면 로직은 최소 변경을 목표로 한다.

## 가시성/명확성
- 응급 알림 중복 표시 방지(최근 `seq` 또는 `caseId` 캐시) 규칙을 유지한다.
